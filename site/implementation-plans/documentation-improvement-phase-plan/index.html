ğŸ“Š Documentation Standards

Writing Style

  • Clear & Concise: Short sentences, active voice
  • Beginner-Friendly: Assume minimal knowledge
  • Practical: Show, don't just tell (examples everywhere)
  • Consistent: Same terminology, formatting, structure
  • Scannable: Headers, bullets, code blocks, callouts

Content Structure

Every guide should have: 1. What: Brief description (1-2 sentences) 2. When: When to use this (use cases) 3. How: Step-by-step instructions 4. Examples: Working code examples 5. Reference: Links to related docs

Code Examples

  • âœ… Complete and runnable
  • âœ… Copy-paste ready
  • âœ… Syntax highlighted
  • âœ… With explanatory comments
  • âœ… Show expected output

Special Elements

Use callouts for: - ğŸ’¡ Tips: Helpful hints - âš ï¸ Warnings: Common pitfalls - âœ… Best Practices: Recommended approaches - ğŸ” Deep Dive: Advanced topics - ğŸ“š See Also: Related documentation
